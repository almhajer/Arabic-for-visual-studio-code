# Security Policy

## Supported Versions

| Version | Supported |
| --- | --- |
| 1.125.x | Yes |
| 1.124.x and earlier | Best effort |

## Reporting a Vulnerability

Please do not open public GitHub issues for suspected security vulnerabilities.

Instead, report them privately through one of the following channels:

- GitHub Security Advisories for this repository
- The repository issue owner contact path if a private advisory cannot be opened

Include:

- A clear description of the issue
- Steps to reproduce
- Affected versions or files
- Any proof-of-concept material that helps confirm impact

## Disclosure Policy

- We will acknowledge receipt as soon as practical.
- We will investigate the report and assess impact.
- We will coordinate a fix before public disclosure when needed.
- We may ask for additional details to reproduce the issue safely.

## Response Expectations

- Initial acknowledgement target: within 7 days
- Status update target: within 14 days when reproduction succeeds
- Remediation timing depends on severity, release timing, and upstream dependency constraints

## Notes

This repository ships localization data and extension packaging metadata. Security reports may still be relevant for supply-chain, packaging, documentation-link, or release-process issues.
