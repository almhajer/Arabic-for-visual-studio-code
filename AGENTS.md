# AGENTS.md

اقرأ هذا الملف أولًا قبل أي تعديل في المشروع.

## تعريف المشروع
هذا المشروع مخصص لتعريب ملفات JSON الخاصة ببرنامج VS Code وإضافاته من اللغة الفرنسية إلى اللغة العربية.

الهدف ليس الترجمة الحرفية، بل إنتاج ترجمة عربية:
- صحيحة معنويًا
- مناسبة للمستخدم النهائي
- متسقة عبر كامل المشروع
- سليمة نحويًا وأسلوبًا
- آمنة بالكامل من ناحية بنية JSON

## الملفات الأساسية
- main.i18n.json
- package.json
- extensions/*.i18n.json

## الهدف التشغيلي
عند العمل على هذا المشروع، يجب:
1. ترجمة النصوص المعروضة للمستخدم فقط
2. الحفاظ الكامل على صحة JSON
3. عدم تغيير المفاتيح أو البنية
4. عدم كسر placeholders أو المتغيرات أو escape sequences
5. اعتماد ترجمة عربية بشرية طبيعية مناسبة لواجهة VS Code
6. الالتزام بالمصطلحات المعتمدة في TERMINOLOGY.md

## مبدأ العمل العام
التعريب في هذا المشروع هو Localization وليس Literal Translation.

لذلك:
- قدّم المعنى الحقيقي
- اجعل النص مناسبًا للواجهة
- فضّل الوضوح على الحرفية
- حافظ على الاتساق الاصطلاحي
- لا تدخل أي تفسير داخل JSON

## قواعد صارمة جدًا

### 1) لا تغيّر بنية JSON إطلاقًا
ممنوع:
- تغيير أسماء المفاتيح
- حذف فاصلة
- إضافة فاصلة خاطئة
- كسر الأقواس `{}` أو `[]`
- تغيير نوع القيم
- تغيير ترتيب البنية ما لم يُطلب ذلك صراحة

### 2) ترجم القيم النصية الظاهرة للمستخدم فقط
ترجم النصوص التي يراها المستخدم فقط، ولا تترجم البنية أو المعرّفات الداخلية.

### 3) ممنوع المساس بالعناصر التالية
اترك هذه العناصر كما هي حرفيًا:
- `{0}`, `{1}`, `{2}`
- `{name}`, `{value}`, `{label}`
- `${variable}`
- `%s`, `%d`, `%f`
- `{{value}}`
- `\n`, `\t`, `\r`
- HTML tags
- Markdown syntax
- النصوص داخل backticks عندما تكون أوامر أو كود
- command ids
- configuration keys
- enum values التقنية
- file paths
- package names
- extension identifiers
- internal ids

### 4) ممنوع الترجمة الحرفية الرديئة
الترجمة المقبولة هي:
- ترجمة طبيعية
- ترجمة وظيفية
- ترجمة واجهات
- ترجمة دقيقة بالسياق

---

## قواعد خاصة بملف package.json
في `package.json`:
- ترجم فقط النصوص الظاهرة للمستخدم
- لا تترجم الحقول التقنية
- لا تترجم ids
- لا تترجم أسماء الأوامر الداخلية
- لا تترجم مفاتيح الإعدادات البرمجية
- لا تعبث بقيم contributes التقنية

يمكن ترجمة أشياء مثل:
- displayName
- description
- title
- command title
- category labels الظاهرة
- configuration descriptions

ولا يجوز ترجمة أشياء مثل:
- name
- publisher
- engines
- activationEvents
- command id
- contributes object keys
- configuration property ids

## قواعد خاصة بملف main.i18n.json
هذا الملف غالبًا يحتوي نصوص واجهة كثيرة جدًا.

المطلوب فيه:
- صياغات عربية واضحة ومختصرة
- الحفاظ على نمط موحد في كامل الملف
- توحيد المصطلحات بين الرسائل المتشابهة
- تجنب الأسلوب الجامد أو الآلي

---

## سياسة المصطلحات
اعتمد TERMINOLOGY.md كمرجع إلزامي.
إذا ظهر مصطلح غير موجود فيه:
1. اختر صياغة عربية واضحة ومتسقة
2. استخدمها في الملف الحالي
3. أضفها لاحقًا إلى TERMINOLOGY.md

---

## أسلوب اللغة العربية المطلوب
يجب أن تكون الصياغة:
- فصيحة وواضحة
- قصيرة إن كانت لزر أو قائمة
- تفسيرية إن كانت لوصف إعداد
- مباشرة إن كانت رسالة خطأ
- طبيعية وغير متكلفة
- موحدة عبر كامل المشروع

### أمثلة أسلوبية
- Configure → تهيئة
- Build running → البناء قيد التشغيل
- Failed to load configuration → فشل تحميل الإعدادات

---

## سياسة التعامل مع الغموض
إذا كانت العبارة الفرنسية تحتمل أكثر من معنى:
1. افهم اسم المفتاح
2. افهم السياق المجاور
3. افهم موضع النص داخل VS Code أو الإضافة
4. اختر الترجمة الأكثر فائدة للمستخدم العربي
5. لا تستخدم ترجمة حرفية مربكة

---

## سياسة الدفعات للملفات الضخمة
إذا كان الملف كبيرًا جدًا:
- لا يتم العمل عليه دفعة واحدة
- يتم تقسيمه إلى دفعات
- كل دفعة تُراجع مستقلًا
- بعد اكتمال كل الدفعات تُجرى مراجعة نهائية شاملة

### حجم الدفعة الموصى به
- من 500 إلى 1500 سطر
أو
- حسب الكائنات أو الأقسام المنطقية داخل JSON

### لكل دفعة يجب تنفيذ الآتي
1. قراءة الجزء كاملًا
2. تحديد ما يُترجم فقط
3. ترجمة القيم ترجمة بشرية
4. فحص placeholders
5. فحص سلامة JSON
6. توحيد المصطلحات مع الأجزاء السابقة
7. إخراج النتيجة فقط

---

## سياسة المراجعة النهائية
بعد اكتمال جميع الأجزاء:
يجب إجراء مراجعة شاملة على كامل الملف للتأكد من:
- توحيد المصطلحات
- عدم وجود ترجمات حرفية رديئة
- عدم اختلاف ترجمة العبارة نفسها بين مواضع متعددة
- سلامة JSON
- الحفاظ الكامل على placeholders
- تناسب النصوص مع واجهة المستخدم

## آلية القرار
عند التعارض بين:
- الحرفية
- الوضوح
- الأسلوب العربي الطبيعي

فالأولوية تكون:
1. سلامة JSON
2. الحفاظ على placeholders
3. صحة المعنى
4. وضوح العبارة للمستخدم
5. الاتساق الاصطلاحي

## ممنوعات
ممنوع:
- ترجمة المفاتيح
- ترجمة identifiers التقنية
- كسر JSON
- حذف نصوص دون سبب
- إدخال شروحات داخل الملف
- استخدام ترجمة حرفية آلية ركيكة
- تغيير placeholders
- ترجمة command ids
- ترجمة configuration keys
- التعديل على القيم التقنية غير الظاهرة للمستخدم

---

## صيغة الإخراج
عند ترجمة JSON:
- أخرج JSON فقط
- دون شرح
- دون Markdown
- دون ملاحظات جانبية
- دون تغيير غير ضروري في التنسيق

عند طلب مراجعة:
- بيّن بوضوح مواضع الخطأ
- حدّد هل المشكلة:
  - معنى
  - اتساق
  - صياغة واجهة
  - placeholders
  - بنية JSON
